# Design Decisions

This folder captures the non-obvious design choices that are easy to lose if they only live in code or scattered prose.

Use these docs when the question is:

- why is the system shaped this way?
- why did we reject the obvious alternative?
- what operational or accounting consequences follow from that choice?
